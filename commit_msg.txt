refactor: 迁 commands/ingestion.rs 6 个内部 helper 到 AppResult + 修 data-cleaner <ol> bug

## Rust 侧

[src-tauri/src/commands/ingestion.rs](src-tauri/src/commands/ingestion.rs) 6 个内部 helper 从 `Result<T, String>` 迁到 `AppResult<T>`：

| Helper | 旧 | 新 |
|--------|------|------|
| `clone_image_processor` | `Result<ImageProcessor, String>` | `AppResult<ImageProcessor>` |
| `extract_image_text_with_processor` | `Result<String, String>` | `AppResult<String>` |
| `create_docx_preview_temp_dir` | `Result<PathBuf, String>` | `AppResult<PathBuf>` |
| `extract_docx_preview_text` | `Result<String, String>` | `AppResult<String>` |
| `extract_text_with_docx_preview_fallback` | `Result<String, String>` | `AppResult<String>` |
| `ingest_file_text` | `Result<IngestionResult, String>` | `AppResult<IngestionResult>` |

Tauri command 边界（`#[tauri::command]` 标记的 `ingest_text`/`ingest_file`/`extract_file_text`/`ingest_directory`）保持 `Result<T, String>`，调用 helper 处统一加 `.map_err(|e: AppError| e.to_string())?` 适配。8 处 match 块错误类型同步转 String。

好处：
- helper 之间能用 `?` 优雅串联
- 锁错误、IO 错误、API 错误、配置错误各有独立 variant
- 前端切换到结构化错误时只翻一行 Tauri command 签名

## TS 侧

**[src/lib/data-cleaner.ts:401-407](src/lib/data-cleaner.ts#L401-L407) 真 bug 修复**

`<ol>` 转换 callback 用 `${i}. $1` 但 callback `() => {...}` 没接捕获组参数，导致有序列表输出字面量 `1. $1\n2. $1\n3. $1`（永远不变）。

修复：callback 改成 `(_m, inner) => \`${i}. ${inner}\``，类型显式声明 `(_outer: string, content: string)` 以满足 strict mode。
