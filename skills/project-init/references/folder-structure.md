# 三级文件夹结构蓝图

项目启动时创建的完整三级目录结构。此蓝图供 `project-init` Skill 和新项目创建时参考。

## 设计原则

- **第一级（阶段）**：对应实施方法论V10.0的7个阶段 + 项目管理 + 被裁剪文件
- **第二级（活动组）**：对应方法论中的活动（Activities），将相近任务归组
- **第三级（交付物类别）**：对应具体的交付物类型或任务组

## 目录总数

| 层级 | 数量 | 说明 |
|------|:----:|------|
| 阶段（一级） | 10 | 00项目管理 + 01~07阶段 + 被裁剪文件 + .claude |
| 活动组（二级） | 31 | 每个阶段2~6个活动组 |
| 交付物类别（三级） | — | 按需创建，部分二级目录即为最细粒度 |

## 与旧版结构对比

| 项目 | 旧结构（V10.0默认） | 新结构（本套件） |
|------|-------------------|----------------|
| 层级 | 2级（阶段/模板） | 3级（阶段/活动组/交付物） |
| 项目管理 | 混入各阶段 | 独立00_项目管理 |
| 模板组织 | 全部平铺在阶段目录 | 按活动组归类 |
| 被裁剪文件 | 无 | 独立目录，保持原路径 |
| 修订版 | 与正式版混合 | 当前版在工作目录，旧版入被裁剪 |
