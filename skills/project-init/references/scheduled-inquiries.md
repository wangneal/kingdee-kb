# 项目主动问询配置

## 创建定时任务

### A. 每周复盘

使用 CronCreate 创建持久化定时任务：
- cron: `0 17 * * 5`（每周五 17:00）
- prompt: 读取 CLAUDE.md 的项目状态和活动日志，回顾本周交付物产出情况，对比总体计划判断进度偏差，列出下周重点工作建议。最后询问用户：「需要我帮你生成周报吗？」
- recurring: true, durable: true

### B. 每日站会

使用 CronCreate 创建持久化定时任务：
- cron: `7 9 * * 1-5`（工作日 9:07）
- prompt: 读取 CLAUDE.md 查看当前阶段和待完成交付物，列出今日重点任务建议。询问用户：「今天有哪些重点工作？是否有阻塞项需要协调？」
- recurring: true, durable: true

### C. 阶段预警

使用 CronCreate 创建持久化定时任务：
- cron: `13 10 * * *`（每天 10:13）
- prompt: 读取 CLAUDE.md 检查各阶段预计完成日期，如有 3 天内即将到期的阶段，列出该阶段交付物完成度（已完成数/总数），提醒用户未完成交付物清单，建议加速推进或申请延期。如果所有阶段都在安全范围内，静默跳过不打扰用户。
- recurring: true, durable: true

## 创建结果展示

```
✅ 主动问询已配置

已开启的定时任务：
  - 每周复盘：每周五 17:00
  - 每日站会：工作日 9:07
  - 阶段预警：每天 10:13（到期前3天自动提醒）

管理：
  - 随时说「配置主动问询」修改或增减任务
  - 说「暂停主动问询」临时关闭
  - 说「查看问询设置」查看当前配置
```

## 管理入口

用户随时可以说以下指令管理主动问询：

| 用户说 | 行为 |
|--------|------|
| "配置主动问询" | 展示当前任务列表，提供增删改选项 |
| "暂停主动问询" | CronDelete 删除所有相关定时任务 |
| "查看问询设置" | CronList 展示当前定时任务及下次触发时间 |
