# 健康度评分公式

> 实现位置：`server/health_score.py`
> 输入：聚合快照 `snapshot`
> 输出：`{score:int, rag:'green|yellow|red', subscores:{...}, trend:int|None}`

---

## 1. 总公式

```
Health = 0.25·Schedule + 0.20·Cost + 0.20·Quality + 0.20·Risk + 0.10·Scope + 0.05·Comm
```

| 子分 | 权重 | 缺数据兜底 |
|------|------|-----------|
| Schedule | 25% | 70 |
| Cost | 20% | 70 |
| Quality | 20% | 80 |
| Risk | 20% | 80 |
| Scope | 10% | 90 |
| Comm | 5% | 70 |

**RAG 阈值**：≥80 绿 / 60-79 黄 / <60 红

---

## 2. 各子分计算

### Schedule（进度）

```
Schedule = clamp(100 - |1 - SPI| × 100, 0, 100)

SPI = 已完成阶段数 / 按计划应完成阶段数
    无计划日期时 → 当前阶段索引 / 7
```

- SPI = 1 → Schedule = 100
- SPI = 0.8 → Schedule = 80
- SPI = 1.2 → Schedule = 80（超前也会扣，避免虚报）

### Cost（成本）

```
Cost = clamp(100 - max(0, 已耗人天/合同人天 - 进度%) × 200, 0, 100)
```

- 缺合同人天 → 70（标注"估算"）
- 已耗人天从 `timesheet.logged` 事件累加
- 进度% = 已完成阶段数 / 7

### Quality（质量）

```
Quality = 100 - min(100, P0×20 + P1×10 + P2×3 + P3×1)
```

- 无缺陷数据 → 80
- P0 缺陷对质量打击最大

### Risk（风险）

```
Risk = 100 - min(100, 高危风险数×15 + 中危×5 + 停滞天数总和)
```

- 无风险数据 → 80
- 停滞天数总和 = Σ(每条 open 风险距今天数)

### Scope（范围）

```
Scope = clamp(100 - 已批准变更人天 / 合同人天 × 300, 0, 100)
```

- 无变更数据 → 90
- 变更人天占合同 10% 时，Scope ≈ 70

### Comm（沟通）

```
Comm = 周报准时率 × 100
```

- 无周报 → 70
- 准时率 = 实际期数 / 应生成期数

## 3. 调权指南

编辑本文件即可调整权重，无需改代码。重启 dashboard 生成即生效。

**常见调权场景**：

| 场景 | 建议调整 |
|------|---------|
| 固定总价合同，成本不敏感 | Cost 权重 20% → 10%，Schedule 25% → 35% |
| 客户对质量极度敏感 | Quality 20% → 30%，Risk 20% → 10% |
| 敏捷迭代，范围常变 | Scope 10% → 5%，Comm 5% → 10% |
| 纯人力外包，无固定里程碑 | Schedule 25% → 15%，Cost 20% → 30% |

**约束**：6 项权重之和必须 = 1.0；单项权重范围建议 0.05 ~ 0.40。

## 4. 真实例子

**项目背景**：
- 合同人天：200
- 已耗人天：90
- 当前阶段：03_方案（第 3 阶段，共 7 阶段）
- 计划应完成：2 阶段
- 缺陷：P0=0, P1=1, P2=2, P3=5
- 风险：高危 1 条（停滞 5 天），中危 2 条
- 变更：已批准 20 人天
- 周报：4 期 / 应 6 期

**计算**：

| 子分 | 过程 | 结果 |
|------|------|------|
| Schedule | SPI = 3/2 = 1.5；\|1-1.5\|×100 = 50；100-50 = **50** | 50 |
| Cost | 进度% = 3/7≈42.9%；90/200=45%；max(0,45%-42.9%)=2.1%；100-2.1%×200≈ **96** | 96 |
| Quality | 100-min(100, 0×20+1×10+2×3+5×1) = 100-21 = **79** | 79 |
| Risk | 100-min(100, 1×15+2×5+5) = 100-30 = **70** | 70 |
| Scope | 100-20/200×300 = 100-30 = **70** | 70 |
| Comm | 4/6×100 = **67** | 67 |

**总分**：
```
Health = 0.25×50 + 0.20×96 + 0.20×79 + 0.20×70 + 0.10×70 + 0.05×67
       = 12.5 + 19.2 + 15.8 + 14.0 + 7.0 + 3.35
       = 71.85 → 72
```

**RAG**：🟡 黄（60-79）

**解读**：进度超前但健康度只有 72，主要被 Schedule 子分（50）和 Risk 子分（70）拉低。建议优先处理停滞风险，并核实进度超前是否伴随质量债务。
