# 双版本周报内容规则

## 概述

周报根据受众分为两个版本：**客户版（甲方版）**和**内部版（项目组版）**。
每一版有严格的包含/排除规则，确保信息安全。

---

## 客户版（甲方版）— 必须排除

以下信息绝不可以在客户版周报中出现：

| 类别 | 具体内容 | 原因 |
|------|---------|------|
| **SOW 内部信息** | SOW版本历史、内部审批状态、SOW差异评审 | 客户只需看到"合同签订进度"，内部 SOW 工作流不可见 |
| **报价/价格** | 报价金额、报价版本迭代、成本测算、人天成本表 | 财务条款属于合同谈判范畴，非周报内容 |
| **人天/成本** | 资源人天及成本表、顾问投入天数、人力成本 | 内部资源规划，对客户透明无意义 |
| **顾问信息** | 除项目经理外的具体顾问姓名、顾问分工细节 | 客户只需知道关键对接人，不需要完整团队名单 |
| **内部审批** | "已完成内部审批"、"等待领导签字" | 客户关心客户方的审批，不关心乙方内部流程 |
| **利润/折扣** | 利润率、折扣率、加价率相关信息 | 商业敏感信息 |
| **乙方内部风险** | 顾问资源复用冲突、内部排期冲突 | 内部产能问题是乙方责任，非客户关注点 |
| **产品限制** | "标准产品不支持XXX"、"需定制开发"等负面表述 | 应表述为"差异分析确认中"或"方案讨论中" |

## 客户版（甲方版）— 应该包含

| 类别 | 具体内容 |
|------|---------|
| **调研进展** | 会议纪要要点、调研覆盖范围、业务发现与洞察 |
| **客户配合事项** | 待确认需求、待提供资料、待安排会议 |
| **共同推进事项** | 合同签订（双方角度）、服务器采购（客户动作+我方响应） |
| **下周计划** | 需要客户参与的会议、需要客户确认的方案 |
| **风险提示** | 影响项目进度的外部因素、需客户决策的事项（软性表述） |
| **里程碑** | 阶段切换节点、方案评审时间、关键成果交付时间 |

## 客户版敏感信息脱敏规则

| 原始表述 | 客户版改写 |
|---------|-----------|
| "SOW V3 已完成内部审批" | "合同条款已进入最终确认阶段" |
| "报价已调整至 85 万人天" | "项目商务条款持续同步中" |
| "李四因资源复用本周仅投入 2 天" | 删除（不出现） |
| "标准产品不支持泰国税率计算" | "泰国税率计算方案讨论中" |
| "等待内部领导签字确认" | 删除（不出现） |
| "顾问张三、李四、王五" | "金蝶项目组" |

---

## 内部版（项目组版）— 必须包含

内部版应包含所有信息，无保留：

| 类别 | 详细程度 |
|------|---------|
| **SOW** | 完整版本历史、客户反馈版本状态、内部审批状态 |
| **报价** | 版本迭代记录、终版版本号、审核状态 |
| **成本** | 人天成本表（按模块/轨道拆分）、资源分配表 |
| **团队** | 完整人员名单含角色：PM、实施顾问、技术架构师、财务顾问 |
| **风险** | 完整风险矩阵（🔴高🟡中⚪已关闭），含影响描述、应对措施、责任人 |
| **问题** | 已关闭问题清单及解决方式 |
| **依赖** | 外部系统集成（GPS、支付平台等）、第三方供应商 |
| **部署** | 服务器采购进度、环境搭建顺序、部署SOP状态 |

## 甲方/乙方人员分类规则

生成客户版时必须区分甲方和乙方人员。

### 常见甲方角色
- 项目负责人 / 项目发起人
- 部门负责人（销售部、财务部、IT部）
- 关键用户
- 高层决策者

### 常见乙方（金蝶）角色
- 项目经理（PM）
- 实施顾问
- 财务顾问
- 技术架构师
- 部署工程师

### 分类原则
1. 从 CLAUDE.md 项目配置中读取金蝶PM → 乙方
2. 从项目通讯录/任命书文件中读取双方名单
3. 不确定时，向用户确认：「XX 是甲方客户方还是乙方实施方？」
4. 客户版中乙方人员统称为"金蝶项目组"（项目经理除外）

## 多轨道项目处理

当项目包含多个并行轨道时（如：泰国租赁 + 埃塞销售 + 集团财务）：

### 轨道检测
- 从项目目录结构中识别独立的模块/领域目录
- 从 CLAUDE.md 项目配置中读取项目范围描述
- 向用户确认轨道划分

### 报告中的轨道处理
- **客户版**：每个轨道独立子章节，待办和风险统一汇总
- **内部版**：轨道特定章节在"工作进展"下，资源分配按轨道拆分

### 轨道命名规范
- 轨道名称应简洁明了：如"泰国租赁业务"、"埃塞销售业务"、"集团财务"
- 内部版中可使用项目内部代号
