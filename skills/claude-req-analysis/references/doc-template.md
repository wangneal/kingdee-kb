# 需求文档模板

根据 Phase 1 选定的风格调整展开方式，但统一包含以下章节：

## 1. 项目概述
- 客户行业
- 项目背景（行业研究 + 客户资料综合）
- 系统目标

## 2. 术语表
| 术语 | 说明 |
|------|------|

## 3. 行业上下文
- 行业现状要点
- 同类系统常见能力
- 合规/标准要求（如适用）

## 4. 功能需求

### 正式规范型
- 使用三级编号（1 / 1.1 / 1.1.1）
- 每个功能含：前置条件、触发条件、处理流程、后置条件、异常处理
- 语言严谨，避免口语化

### 敏捷用户故事型
- 核心格式：As a [角色], I want [功能], So that [价值]
- 每条附验收标准（Given/When/Then）
- 可按 Epic → Story 组织

### 产品 PRD 型
- 按用户场景组织
- 含交互说明（流程、状态变化）
- 含异常和边界情况

### 简洁清单型
- 表格为主：功能点 | 描述 | 优先级 | 来源
- 适合快速对齐
