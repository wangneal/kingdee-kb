# 顾问工作台 — 终端用户使用说明书

> 软件名：顾问工作台（内部代号 KingdeeKB）
> 适用版本：v0.1.x

---

## 一、产品定位

KingdeeKB 是为金蝶 ERP 实施顾问打造的桌面 AI 助手，提供：

- **本地知识库**：把项目文档、合同、调研记录等沉淀在本地，AI 基于真实内容回答
- **AI 对话与交付物生成**：从一句话问题到一份调研报告，AI 帮你完成
- **项目化工作台**：把客户项目作为组织单元，按项目隔离知识、数据和风险

应用为离线优先的桌面程序（基于 Tauri），数据保存在本机 SQLite 中，仅 LLM 调用走云端/本地模型。

---

## 二、安装与启动

### 2.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS 12+、主流 Linux 发行版 |
| 内存 | 8 GB 以上（首次加载本地嵌入模型需要 4 GB） |
| 磁盘 | 至少 5 GB 可用空间（含嵌入模型缓存） |
| 网络 | 仅在使用在线 LLM/ASR 时需要 |

### 2.2 安装

从发布渠道获取安装包（`.exe` / `.dmg` / `.AppImage` / `.deb`），按系统提示安装即可。

### 2.3 首次启动

应用启动后，会在后台静默加载本地嵌入模型（BGE-Small-ZH，约 90 MB）。首次启动需等待数秒至 1 分钟，左下角状态栏会显示 Embedding 加载进度。

## 三、初始配置

进入「设置」页面（侧边栏底部齿轮图标），按需配置以下项目。

### 3.1 LLM 服务商

**推荐**：使用云端 OpenAI 兼容接口（OpenAI、DeepSeek、通义千问、智谱、月之暗面等）或本地 Ollama。

配置项：
- **API 地址**：服务商提供的 base URL（OpenAI 兼容协议）
- **API Key**：点击「生成」自动写入系统钥匙串，不会明文保存到磁盘
- **模型名称**：如 `gpt-4o-mini`、`deepseek-chat`、`qwen2.5:14b`
- **温度参数**：建议 0.2-0.5，过高会导致回答不稳定

> 至少配置一个供应商并通过「测试连接」后，才能使用 AI 对话、调研助手、风险把控等功能。

### 3.2 嵌入模型（Embedding）

首次使用知识库时，应用会自动从本地缓存目录加载 `bge-small-zh-v1.5` 模型。如未下载，可点击「下载模型」按钮（设置页 → Embedding）。

### 3.3 语音识别（可选）

如需在调研助手中使用语音输入，可选择：
- **本地 Whisper**：免费、离线，但占用磁盘较大
- **腾讯云 ASR**：在线 API，识别准确率高
- **讯飞 ASR**：在线 API，中文优化好

### 3.4 腾讯会议 MCP（可选）

在「设置 → 腾讯会议 MCP」填入 [meeting.tencent.com/ai-skill](https://meeting.tencent.com/ai-skill) 获取的 Token，即可启用「会议管理」页的全部能力：

- **预约会议**：填主题+开始/结束时间，一键在腾讯会议创建会议并获取会议号
- **查询会议详情 / 成员 / 等候室 / 受邀人**
- **取消 / 修改会议**
- **同步转写 + AI 智能纪要**：选择已结束会议 → 一键拉取转写全文与官方 AI 纪要
- **录制权限申请**：遇到无权限错误时按提示走两步式申请流程
- **Agent 意见箱**：遇到工具错误/能力不足时通过 `submit_feedback` 上报

侧边栏「腾讯会议客户端内的悬浮助手」仍可继续使用——它通过 localStorage 桥接把转写文本送进 KingdeeKB 调研助手。

---

## 四、项目切换

侧边栏顶部为「项目切换器」。KingdeeKB 以「项目」为单位组织所有数据：

- 不同项目的知识库、调研会话、风险记录、产物完全隔离
- 切换项目后，所有页面的数据上下文会立即更新
- 「概览」页会显示当前项目的关键指标

**首次使用建议**：在「项目管理 → 详情」中创建第一个客户项目，填写客户名称、合同范围、当前阶段等信息。

## 五、功能模块

### 5.1 概览（Home）

进入应用后的默认页面，提供：

- **项目仪表盘**：当前项目的文档数、产物数、阶段进度
- **阶段跟踪**：项目的实施阶段列表，可快速跳转当前阶段
- **快捷入口**：常用功能（知识库、检索、AI 对话、调研、风险）的快捷卡片

### 5.2 知识库（Browse）

项目内所有已导入文档的浏览入口。

- **文档树**：按文件夹/标签组织的文档列表
- **文档详情**：右侧显示文档元数据、Markdown 渲染、关联的 Wiki 页面
- **右键菜单**：支持「在 AI 对话中提问」「提取为 Wiki 页面」「删除」等
- **搜索框**：顶部搜索基于全文检索 + 向量检索的混合结果

> 文档导入入口已移至知识库内的「导入文档」按钮（在空状态或工具栏）。详细导入流程见 §5.3。

### 5.3 文档导入（Import）

支持多种导入方式：

- **单文件导入**：拖拽或选择单个文件
- **文件夹批量导入**：选择整个文件夹，自动递归扫描
- **腾讯会议记录导入**：从已对接的腾讯会议 MCP 拉取会议记录

**支持的文件格式**：

| 类型 | 格式 |
|------|------|
| 文本 | `.md`、`.txt`、`.text`、`.markdown`、`.html`、`.htm` |
| Office | `.pdf`、`.doc`、`.docx`、`.xls`、`.xlsx` |
| Visio | `.vsd`、`.vsdx` |
| 视频 | `.mp4`、`.webm`、`.avi`、`.mov`、`.mkv`、`.flv`、`.wmv`（自动转音频后调用 ASR） |
| 音频 | `.m4a`、`.mp3`、`.wav` |
| 图片 | `.png`、`.jpg`、`.jpeg`、`.gif`、`.bmp`、`.webp`（通过视觉模型理解） |

导入后，应用会执行切片、嵌入、索引三步处理：
1. 提取文本/转写音视频
2. 按段落切片（带重叠）
3. 计算向量并写入 SQLite + 索引

处理进度可在「项目管理 → 队列」中查看。

### 5.4 检索（Search）

专门的混合检索页面，比知识库内嵌搜索功能更全：

- **查询输入框**：支持自然语言或关键词
- **结果列表**：显示匹配片段、来源文档、相关度评分
- **筛选器**：按文档类型、项目、日期范围过滤
- **结果操作**：点击「在 AI 对话中追问」可直接基于该片段提问

### 5.5 AI 对话（Chat）

核心对话界面。基于 ReAct Agent 的多轮对话：

- **多轮上下文**：自动维护对话历史，支持连续追问
- **附件支持**：粘贴图片、上传文件作为参考依据
- **流式输出**：实时显示 AI 思考过程和工具调用
- **澄清提问**：AI 不确定时会主动弹出澄清问题
- **快捷操作**：`Enter` 发送、`Shift + Enter` 换行、点击「清空对话」重新开始

**典型用法**：
- 询问项目相关的实施问题
- 让 AI 基于上传的合同/截图分析风险
- 让 AI 生成周报、会议纪要初稿

### 5.6 调研助手（Research Assistant）

结构化调研记录工具。

**主要流程**：
1. **创建调研会话**：选择调研模块、填写访谈对象和日期
2. **逐条记录问答**：手动输入或基于知识库自动生成参考答案
3. **语音输入**：支持录音转文字（需配置 Whisper 或在线 ASR）
4. **蓝图提炼**：一键从调研记录中提炼 As-Is（现状）/ To-Be（目标）/ Gap（差距）
5. **导出报告**：导出为 Markdown 或 CSV

**辅助能力**：
- 思维导图视图：把调研记录以大纲/脑图形式呈现
- 知识库联动：记录时自动检索相关知识库片段
- 腾讯会议导入：从会议记录生成调研问答

### 5.7 风险把控（Risk Control）

双轨风险管控工具。

**核心功能**：

- **需求蔓延检测**：上传合同或选择已导入合同文档，AI 自动提取「合同内」「合同外」清单；后续收到的新需求可一键比对是否在合同范围内
- **项目健康度评估**：4 维度评分（缺席率 / 数据延迟 / 问题积压 / 配合度），从项目数据和交互记录中自动计算
- **防身话术生成**：针对常见困境（如客户拖延、范围争议、验收推诿）生成沟通话术
- **AI 深度分析**：与 AI 对话讨论风险策略
- **数据备份恢复**：项目范围、健康度等数据的导出和导入

### 5.8 产物管理（Products）

所有由技能系统生成的交付物（如调研报告、蓝图、周报、验收包等）统一在此管理：

- **按项目查看**：自动按当前项目过滤
- **类型筛选**：按交付物类型（DOCX / PPTX / MD）筛选
- **导出/打开**：双击在系统默认应用中打开，或导出到任意位置
- **删除**：支持删除产物文件

### 5.9 项目管理（Projects）

项目的管理中心（区别于概览中的项目仪表盘）。

**主要 Tab**：

- **详情**：编辑项目名称、客户、合同范围、当前阶段等元数据
- **阶段**：维护实施阶段列表（计划日期、实际日期、负责人）
- **源数据**：管理项目下已导入的原始文档
- **队列**：查看文档导入处理进度，可重试失败任务

### 5.10 会议管理（Meetings）

腾讯会议 MCP 的前端入口（需先在「设置」配置 Token）。与「调研助手」互补：调研助手面向"事后转写"，本页面面向"全生命周期管理"。

**能力**：

- **预约会议**：弹窗输入主题/开始/结束时间，自动调用 `convert_timestamp` 归一化时区，再调 `schedule_meeting` 创建会议，返回会议号
- **未开始 / 进行中会议列表**：来自 `get_user_meetings`
- **历史会议列表**：来自 `get_user_ended_meetings`，按时间倒序
- **会议详情**：从 `get_meeting` 拉取，附"在腾讯会议中打开"快捷链接
- **同步转写 + AI 智能纪要**：调 `fetch_tencent_meeting_transcript`（已封装 `get_records_list` → `get_transcripts_details` → `get_smart_minutes`）
- **取消会议**：调 `cancel_meeting`（执行前会要求二次确认）

**典型用法**：

1. 在 AI 对话中说"明天下午 3 点和客户开 1 小时的需求确认会"→ Agent 自动调 `convert_timestamp` + `schedule_meeting`
2. 会议结束 30 分钟内打开「会议管理」→ 切换"已结束"→ 选中会议 → "同步转写 + 智能纪要"→ 把转写贴到 AI 对话生成结构化纪要

**未实现项**（已记录到 ARCHITECTURE §8）：

- 会议自动定时同步（当前需手动点击）
- 纪要自动落盘到 `00_项目管理/会议纪要/` 目录（当前需在 AI 对话中执行）
- 待办自动写入活动日志

### 5.11 技能体系（Skills）

可扩展的外部技能框架。KingdeeKB 内置 25+ 个开箱即用的技能：

- **调研类**：调研助手、需求分析、根因分析
- **交付物类**：调研报告、蓝图、PPT、上线包、验收包、会议纪要、周报
- **质量类**：数据清洗、文档脱敏、人性化改写
- **管理类**：变更管理、风险管理、测试管理、调研大纲

**用法**：在 AI 对话中描述需求，Agent 会自动匹配并执行相关技能；产物自动登记到「产物管理」。

**自定义技能**：支持导入符合 `SKILL.md` 规范的技能包（解压到 `skills/` 目录即可）。

### 5.11 知识图谱（Knowledge Graph）

基于已导入文档构建的实体关系网络：

- **节点类型**：客户、产品、模块、人员、流程
- **关系类型**：属于、依赖、关联、负责人
- **视图切换**：支持力导向图、树状图、列表视图
- **统计面板**：展示节点/关系数量、覆盖率、推荐补全

### 5.12 设置（Settings）

所有可配置项的集中入口，分区组织：

- **LLM 供应商**：增删改供应商、API Key、模型、温度
- **Embedding**：模型下载、加载状态、缓存清理
- **语音识别（ASR）**：Whisper / 腾讯云 / 讯飞配置
- **腾讯会议 MCP**：连接凭证配置
- **数据管理**：项目数据导入/导出、本地缓存清理
- **快捷键**：查看和自定义全局快捷键（默认 `Alt + Space` 唤起 Spotlight）
- **关于**：版本信息、问题反馈渠道

## 六、典型工作流

### 6.1 新项目启动

1. 进入「项目管理」创建项目，填写客户名、合同范围
2. 在「详情」中导入合同、调研计划等文档
3. 等待文档处理完成（队列清空）
4. 在「概览」查看项目仪表盘

### 6.2 调研阶段

1. 进入「调研助手」，创建调研会话
2. 选择模块（如「采购管理」），逐条记录客户问答
3. 使用语音输入提速
4. 调研完成后，点击「蓝图提炼」生成 As-Is/To-Be/Gap
5. 导出为 Markdown 进入交付物环节

### 6.3 风险管控

1. 进入「风险把控 → 需求蔓延」
2. 上传合同或选择已导入的合同
3. AI 自动提取「合同内 / 合同外」清单
4. 收到新需求时，新增条目并触发对比
5. AI 标注是否在合同范围内，并给出风险说明

### 6.4 周报生成

1. 进入「AI 对话」
2. 输入「生成本周周报」
3. AI 自动调用 `weekly-report` 技能，扫描项目数据
4. 生成 Markdown 周报，确认后导出或登记到产物管理

## 七、常见问题

### Q1：AI 回答不准确怎么办？

- 确认知识库中已导入相关文档（首页可见文档数）
- 在「知识库」或「检索」中验证是否能找到相关片段
- 调整温度参数（设置页 → LLM）：越低越精确，越高越发散
- 在对话中明确指出「请基于知识库中的 XX 文档回答」

### Q2：导入的文档搜索不到内容？

- 在「项目管理 → 队列」中确认文档已处理完成
- 处理失败的文档可点击「重试」
- 长文档的检索质量取决于切片粒度（默认按段落）

### Q3：应用卡住了怎么办？

- 等待 10 分钟 — Agent 会话有自动超时保护
- 点击输入框旁的取消按钮中断当前操作
- 如仍无响应，强制关闭并重新打开（数据会自动保存）

### Q4：如何更新应用？

- 应用内会自动检查更新（如启用）
- 也可从发布渠道下载最新版本手动安装
- 数据保存在本地目录 `%APPDATA%/kingdee-kb/`（Windows）或 `~/Library/Application Support/kingdee-kb/`（macOS）

### Q5：数据存储在哪里？会泄露吗？

- 所有项目数据、对话历史、知识库、产物文件均存储在本地
- 仅 LLM 调用（对话/嵌入/ASR）会通过网络发送必要数据
- 建议使用本地 Ollama 模型实现完全离线

### Q6：API Key 存放在哪里？

- 通过「生成」按钮配置的 Key 会写入操作系统的钥匙串（Windows Credential Manager / macOS Keychain / Linux Secret Service）
- 不会以明文形式保存到配置文件或日志中

### Q7：能否多人协作？

- 当前版本（v0.1.x）为单用户本地工具
- 多人协作需通过「数据管理 → 导出」共享项目包

## 八、反馈与支持

- **问题反馈**：在 AI 对话中描述复现步骤和环境信息，记录在调研会话中作为待办
- **数据备份**：定期通过「设置 → 数据管理 → 导出」备份项目包，避免本地数据丢失
