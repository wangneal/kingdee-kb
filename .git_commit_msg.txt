fix: 启动 panic — Tauri setup 中改用 tauri::async_runtime::spawn

## 现象
应用启动时 panic:
  thread 'main' panicked at src\app_state.rs:294:13:
  there is no reactor running, must be called from the context of a Tokio 1.x runtime

## 根因
AppState::new() 在 Tauri 的 setup 同步钩子中调用。
此阶段 tokio 运行时尚未初始化（runtime 在 .run() 进入事件循环后才建立），
tokio::spawn 找不到当前线程 runtime，直接 panic。

## 修复
改用 tauri::async_runtime::spawn：
- Tauri 自带 runtime，事件循环启动后由框架统一管理
- 不依赖全局 tokio runtime 状态
- 在 setup 同步钩子中调用安全可靠

## 验证
- cargo check：通过
- 启动栈帧：AppState::new → tokio::spawn → spawn_inner 已改为
  AppState::new → tauri::async_runtime::spawn，再无 panic
