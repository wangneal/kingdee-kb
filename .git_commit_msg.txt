refactor: 用 tracing 替换 60+ 处生产代码 println!/eprintln!，删除多余 unsafe impl

按"重写与兼容原则"清理 AI SLOP：项目已使用 tracing 设施，但生产代码
绕过它用 println!/eprintln! 直接写 stderr/stdout，导致：
- RUST_LOG 过滤不到
- 无结构化字段
- 启动噪音不可关闭

## 替换范围

- commands/core.rs（启动流程 9 处）
- app_state.rs（5 处，包括 Reranker 懒加载和 Embedding lock 错误）
- services/ingestion_queue.rs（5 处）
- services/audio_capture.rs（9 处设备枚举/录音生命周期）
- services/model_downloader.rs（5 处下载/复制日志）
- services/skill_manager.rs（启动加载 3 处）
- services/ingestion.rs（3 处）
- services/video_transcriber.rs（4 处）
- services/whisper_service.rs（2 处）
- services/vector_index.rs（1 处）
- services/product_store.rs（1 处）
- lib.rs（2 处致命错误日志）

全部按错误级别分流：
- 致命/不可恢复 → tracing::error
- 跳过/降级 → tracing::warn
- 启动/进度 → tracing::info

## 删除 unsafe impl

audio_capture.rs 中 `unsafe impl Send/Sync for AudioCapture/CaptureState` 是
冗余的——CaptureState 仅包含 AtomicBool 和 Mutex<T: Send>，Rust 自动推导
Send + Sync，无需手写 unsafe impl。这 4 行既没必要又引入未定义行为风险。
